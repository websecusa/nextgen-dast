{# Password handling on Re-scan. When the source assessment has a stored password, we display a fixed-length asterisk sentinel in the field as a visual cue that credentials are on file. The real password is NEVER echoed into the DOM -- on submit the server reads the hidden prefill_creds_from token (above) and resolves the stored value, but only after verifying the source assessment's FQDN matches the FQDN being scanned (so a tampered token cannot pull another target's creds). The user can: - leave the sentinel intact -> use the stored password, - clear the field -> anonymous re-scan, - type a new password -> override the stored one. #}
