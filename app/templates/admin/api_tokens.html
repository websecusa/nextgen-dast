API tokens

OUI-format bearer tokens for the REST API. Each token is restricted to a whitelist of source IPs / CIDR ranges; requests from any other source are rejected with 403. The secret half of a token is shown only once at creation time and is stored as a SHA-256 hash, so a database leak never yields a usable credential.

Quick reference

  • /api/v1/docs — Swagger UI playground (live, calls the running API).
  • /api/v1/openapi.json — OpenAPI 3.0 service definition.
  • /api/v1/postman.json — Postman v2.1 collection (import into Postman).
  • /api/v1/health — liveness probe (no auth).
