AI Prompts (Enhanced-AI-Testing)

These prompts drive the advanced-tier post-scan LLM pass. Each row is one scenario — the orchestrator iterates the active rows in sort order, evaluating each scenario's fire_when expression against the per-assessment telemetry summary and skipping silently when it evaluates false (so unused scenarios burn no LLM budget).

Editing a prompt's body is a security-sensitive operation — the system-prompt boilerplate carries the non-destructive operating constraints. Per-prompt edits land in the audit log; Restore to default reverts a seeded row to its in-code baseline.
