Master switches

SSO mode

Enable SAML 2.0 turns on the /saml/* endpoints and adds a "Sign in with SSO" button to the login page. Force SAML only also disables local password login — except when the bypass file /data/.saml_bypass exists on the host, which re-enables /login until you remove the file. Always test SSO with a local backup admin signed in elsewhere before flipping force on.

IdP vendor

Picking Use Okta changes the field labels and placeholders below to match the names Okta uses in its "View SAML setup instructions" panel. The wire protocol is standard SAML 2.0 either way — this only affects what the form looks like.

IdP details

Paste these from your Okta application's Sign On → SAML Signing Certificates → View SAML setup instructions panel (or your IdP's equivalent). All four fields are required.

SP details (configure in Okta)

These three values describe nextgen-dast to your IdP. The defaults below are auto-filled from this install's hostname so you can copy them straight into Okta's app wizard. Leave the inputs blank to use the auto-derived values; override only when a non-standard proxy mangles the request host.
