Toolkit — validation probes

Each probe takes an unvalidated finding and produces a confirmed or rejected verdict with concrete evidence. Probes are standalone CLI tools — a pentester can pull any probe out of the container and run it directly. The orchestrator + LLM consolidation phase will wire these in automatically (next pass).

Standalone use (any pentester, any target)

docker compose exec nextgen-dast python /app/toolkit/probes/sqli_boolean.py \
  --url 'https://target.example.com/products?id=1' \
  --param id \
  --cookie 'session=abc123' \
  --scope target.example.com \
  --max-requests 12

All probes accept --stdin for orchestrator-driven JSON input, and --dry-run to print what they would do without sending requests.
