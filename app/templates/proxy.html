How to test

The proxy is a reverse proxy: clients connect to it directly (no proxy settings on the client). With the defaults, point your scanner / browser at:

https://{{ state.proxy.listen_host }}:{{ state.proxy.listen_port }}/

It will be re-issued to {{ state.proxy.upstream }} with the Host header rewritten to {{ state.proxy.upstream_host_header or "(unchanged)" }}. Every request and response is dumped to /data/flows/<testid>_request.txt and /data/flows/<testid>_response.txt on the host.

The proxy presents mitmproxy's CA cert. Import ~/.mitmproxy/mitmproxy-ca-cert.pem from inside the container (or copy it out via docker cp) into the test client trust store to avoid TLS warnings.
